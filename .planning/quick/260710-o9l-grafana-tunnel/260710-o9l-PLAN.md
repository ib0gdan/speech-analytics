Design facts established by live measurement during planning (do not re-derive, but DO re-prove
with the verify commands):
- Grafana's `<base href="/grafana/">` + relative assets = host-agnostic rendering. The only
  absolute redirect Grafana emits is on the BARE `/` path (301 -> root_url). Through the tunnel,
  nginx routes `/` to OpenWebUI, so Grafana's bare-`/` redirect is never reached by tunnel users;
  they hit `/grafana/` directly. This is why a hardcoded root_url host is safe.
- Recommended `GF_SERVER_ROOT_URL=http://localhost:3001/grafana/`: 3001 is the host-mapped direct
  port, so a stale bare-`:3001/` bookmark 301-self-heals to `:3001/grafana/` (which works). The
  host is cosmetic for the tunnel (proven by the foreign-Host test). If the executor's foreign-Host
  proof fails for any reason, the fallback is the same value with a different host — but it must NOT
  hardcode a trycloudflare hostname (it changes every restart).
- The proxy already re-resolves upstreams per request (`resolver 127.0.0.11` + variable) because
  nginx caching an old IP caused silent 502s. Reuse that exact pattern for grafana so an ABSENT
  (profiled-off) grafana simply 502s on `/grafana/` while nginx starts fine. Do NOT add
  `depends_on: grafana` to `proxy` — grafana has `profiles: ["metrics"]`, and a non-profiled service
  depending on a profiled one breaks plain `docker compose up -d`.
